**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/)

**Week 2:** [Setting up custom network infrastructure with Amazon VPC and configuring VPC Peering](1.2-week2/)

**Week 3:** [Deploying EC2 instances, EBS volumes, managing resources via AWS CLI, and coding on Cloud9](1.3-week3/)

**Week 4:** [Securing EC2 with IAM Roles, S3 Static Web Hosting, and global content delivery via CloudFront & Lambda@Edge](1.4-week4/)

**Week 5:** [Deploying database solutions on AWS: Amazon RDS, DynamoDB, and ElastiCache Redis](1.5-week5/)

**Week 6:** [Deploying and containerizing cost-effective applications with Amazon Lightsail and Lightsail Containers](1.6-week6/)

**Week 7:** [Configuring EC2 Auto Scaling, monitoring with CloudWatch, and managing DNS with Route 53](1.7-week7/)

**Week 8:** [Administering Windows Server, AWS Managed Microsoft AD, and building High-Availability Web architecture](1.8-week8/)

**Week 9:** [Requirements Analysis and Architecture Design for Web IT Coach Project](1.9-week9/)

**Week 10:** [Core Web Feature Development and RDS & S3 Database Connectivity](1.10-week10/)

**Week 11:** [Finalizing Web Features, Optimization, and Deploying with ALB & ASG](1.11-week11/)

**Week 12:** [End-to-End Testing, Bug Fixing, Report Writing, and Workshop Preparation](1.12-week12/)
